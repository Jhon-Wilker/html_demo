Como criar um Commit

Adicionando as alterações:

Um Commit é um pacote de alterações feitas no repositório. Cada commit possui arquivos alterados, autor e uma mensagem de resumo.

Para criar um commit, primeiramente deve ser utilizado o comando:

# git add .

O "." irá salvar todas as alterações feitas no arquivo. Caso queira especificar, pode-se utilizar o nome do arquivo o qual as alterações foram feitas no lugar.

Verificando o status do repositório:

Após adicionar as alterações, o seu commit já está pronto para ser criado, mas caso queira, pode verificar o estado de seu repositório.

O repositório pode ser verifcado utilizando o comando:

# git status

Este comando sempre retornará o atual estado do repositório e o que pode ser realizado a seguir, muito útil para visualizar quais arquivos serão alterados.

Criando a commit:

Agora que suas alterações foram salvas, você está pronto para embala-las em um commit.

Para criar um commit pode-se utilizar o comando:

# git commit -m "Descrição do que foi alterado"

Após isso o seu commit já está pronto para ser enviado!
